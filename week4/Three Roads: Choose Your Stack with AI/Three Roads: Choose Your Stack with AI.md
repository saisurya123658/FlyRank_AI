# Three Roads: Choose Your Stack with AI

## My Constraints

* **Budget:** Free tools and hosting only.
* **Current Skill Level:** Intermediate. I know HTML, CSS, JavaScript, React.js, Java, Node.js, Express.js, MySQL, PostgreSQL, and Git. I am still learning advanced backend development.
* **Portfolio Needs:** My portfolio should display:

  * Home
  * About Me
  * Skills
  * Projects
  * Certificates
  * Resume
  * Contact
  * GitHub and LinkedIn links

  My projects include image galleries, GitHub repositories, project descriptions, live demos (where available), and downloadable resume.
* **Display Requirements:**

  * Image galleries for certificates and project screenshots
  * GitHub repository links
  * Live demo links
  * Long-form project descriptions
* **Backend Requirement:** Not yet. A database or authentication is not necessary at this stage.

---

## Option 1 – Static Portfolio (Simplest)

**Stack**

* HTML
* CSS
* JavaScript

**Hosting**

* GitHub Pages (Free)

**Backend**

* No

**How I Would Build**
Create a multi-page responsive website using HTML, CSS, and JavaScript. Projects, certificates, and resume would be displayed as static content.

**Trade-offs**
Very easy to build and maintain, but adding new features or interactive components later requires more manual work.

---

## Option 2 – React Portfolio (Chosen)

**Stack**

* React.js
* React Router
* CSS
* GitHub Pages or Vercel (Free)

**Backend**

* No

**How I Would Build**
Build a single-page React application with reusable components for Home, About, Skills, Projects, Certificates, Resume, and Contact sections. Projects will include GitHub links, screenshots, and live demos.

**Trade-offs**
Slightly more complex than plain HTML, but much easier to organize, update, and expand. It demonstrates modern frontend development skills that employers expect.

---

## Option 3 – Full Stack Portfolio (Most Powerful)

**Stack**

* React.js
* Node.js
* Express.js
* PostgreSQL
* Vercel (Frontend)
* Render/Railway (Backend)

**Backend**

* Yes

**How I Would Build**
Develop a React frontend connected to a Node.js backend with PostgreSQL. Include dynamic project management, contact form storage, authentication, and an admin dashboard.

**Trade-offs**
Most flexible and scalable, but requires significantly more setup, deployment, debugging, and ongoing maintenance.

---

## Pressure Testing the Options

### What breaks if I choose the simplest option?

A static website is enough for displaying my work, but it is harder to maintain as the portfolio grows. It also demonstrates fewer modern development skills.

### What do I maintain if I choose the most powerful option?

I would need to maintain the frontend, backend, database, hosting services, APIs, and deployments. This adds complexity that my current portfolio does not require.

### Can I finish in two weeks?

* Static Portfolio: Yes
* React Portfolio: Yes
* Full Stack Portfolio: Probably not if I want a polished result.

### Does it display my work properly?

* Static Portfolio: Yes
* React Portfolio: Yes
* Full Stack Portfolio: Yes

All three options can display images, GitHub repositories, live demos, and project descriptions effectively.

---

# Final Decision

I chose **React.js with GitHub Pages (or Vercel)** because it provides the best balance between simplicity and modern development practices. It is completely free, easy for me to maintain, and showcases my frontend skills while allowing me to organize projects using reusable components. It also displays my work well through project screenshots, GitHub repositories, live demos, certificates, and detailed descriptions.

I did not choose the static HTML/CSS/JavaScript option because it becomes harder to maintain as my portfolio grows. I also did not choose the full-stack React + Node.js + PostgreSQL option because my portfolio does not currently require a backend, and maintaining a server and database would add unnecessary complexity. I believe the React-based approach is the most practical choice that I can complete within two weeks while creating a professional portfolio.
